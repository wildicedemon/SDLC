# Security Architecture

Research on defensive architecture for autonomous AI coding systems, including prompt injection defense, context poisoning mitigation, MCP privilege isolation, secure inter-agent communication, sandboxing, egress controls, adversarial testing, and anti-hallucination safeguards.

## Core Artifacts

- [overview.md](./overview.md) — Executive summary, framing, synthesis, and open questions
- [comparisons.md](./comparisons.md) — Comparative tables across security approaches
- [patterns.md](./patterns.md) — Security patterns and anti-patterns
- [references.md](./references.md) — Structured references with metadata and tags

## Key Questions

1. Which layered defenses best reduce prompt injection success rates in tool-using agents?
2. How should MCP permissions and trust boundaries be modeled to contain compromise?
3. What execution sandbox and network controls are practical for agentic SDLC pipelines?
4. Which anti-hallucination strategies provide robust protection without excessive latency?
