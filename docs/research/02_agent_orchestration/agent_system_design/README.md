# Agent System Design

## Description

Agent System Design encompasses the architectural principles, patterns, and mechanisms for constructing reliable, scalable AI agents in autonomous coding contexts. This topic covers agent operational modes, skill management, workflow orchestration, rules and guardrails, tool use strategies, CLI and IDE integrations, multi-agent coordination, lifecycle management, failure detection and recovery, and performance evaluation mechanisms.

## Research Artifacts

| File | Description |
|------|-------------|
| [overview.md](overview.md) | Executive summary, topic framing, subtopic synthesis, prior research integration, and open questions |
| [comparisons.md](comparisons.md) | Comparative tables for agent frameworks, failure recovery approaches, and monitoring systems |
| [patterns.md](patterns.md) | Identified patterns and anti-patterns in agent system design |
| [references.md](references.md) | Full structured source list with metadata and citations |

## Key Subtopics

- **Modes**: Agent operational modes and switching mechanisms
- **Skills**: Agent skill definitions, loading, and management
- **Workflows**: Structured multi-step agent workflows
- **Rules**: Agent rules, constraints, and guardrails
- **Tool Use Strategy**: Tool alignment with user intent
- **CLI-Based Agents**: Kilo CLI, Cline CLI, GitHub Copilot CLI, Augment CLI
- **IDE-Integrated Agents**: VS Code Agent Mode, Kilo, Cline, Augment
- **Multi-Agent Orchestration**: Hierarchical patterns, swarming, specialization
- **Lifecycle Management**: State machines, watchdog monitoring
- **Failure Recovery**: Deadlock/livelock detection, graceful degradation
- **Performance Scoring**: Trust scoring, voting thresholds, consensus mechanisms

This topic covers agent operational modes, skill management, workflow orchestration, rules and guardrails, tool use strategies, CLI and IDE integrations, multi-agent coordination, lifecycle management, failure detection and recovery, and performance evaluation mechanisms.
