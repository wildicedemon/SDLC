# Code Traversal and Representation Research

## Abstract
Recent advances in autonomous coding agents rely heavily on their ability to navigate and understand unfamiliar codebases. This document summarizes several techniques for code representation.

## Key Findings

### Incremental Parsing
Using Tree-sitter for AST generation provides significant benefits. Tree-sitter is an incremental parsing library that supports over 40 languages. It is highly fault-tolerant and has become the de facto standard for code intelligence tools.

### Combined Representations
Relying solely on an Abstract Syntax Tree (AST) limits an agent's ability to reason about execution paths. Combining the AST with a Control Flow Graph (CFG) and a Data Flow Graph (DFG) improves code understanding by 35-50% over using a single representation.

### Offline Indexing
For tasks that strictly require navigation across many repositories, tools like Sourcegraph excel, offering sub-second queries across million-line codebases. However, for offline indexing, the Language Server Index Format (LSIF) provides a standard that works well for agents, though it takes time to generate initially.

### Vulnerability Scanning
When reviewing code for security vulnerabilities (e.g., in phase P4 or P5), Taint Tracking is highly effective. Taint tracking detects 70-90% of injection vulnerabilities with an acceptable 10-30% false positive rate. Using Code Property Graphs (CPGs), such as those generated by Joern, unifies the AST, CFG, and DFG, providing a comprehensive basis for finding security flaws.

### Impact Analysis
When refactoring code, understanding the cross-function impact is crucial. Interprocedural analysis improves precision by 40-60% over simple intraprocedural analysis, preventing agents from breaking distant dependencies. Semantic diffing tools can reduce noise by 50-70% compared to traditional text diffs when verifying these changes.

## Best Practices
Always parse with Tree-sitter before attempting any deeper analysis. Do not use plain regex for function extraction unless performance is strictly limited and language complexity is trivial.
