# Technical Specification

## Technical Context
The repository currently contains scattered research and distillation artifacts generated by multiple agents over time. These documents include various markdown files, JSON outputs, and research logs spread across several directories (`distillation/`, `distilled/`, `docs/distillation/`, `docs/distilled/`, `docs/research/`, `Kimi-Research/`, and root directories). 
Among these, `docs/distillation/master_index.md` represents a highly structured and comprehensive consolidation of this research (covering 372 knowledge atoms, 12 domains, 8 phases, 47 products).
The objective is to clean up, consolidate, and organize this research without modifying any operational scripts or code (`.py`, `.ts`, scripts/, etc.).

## Implementation Approach
1. **Identify the Source of Truth:** Establish `docs/distillation/` as the primary canonical repository for all research distillation.
2. **Consolidate and Archive Redundant Artifacts:** Move older or redundant research directories (`distillation/`, `distilled/`, `docs/distilled/`, and scattered root research markdown files) into a designated archive directory (e.g., `docs/research_archive/`) to avoid clutter and confusion.
3. **Verify Research Completeness:** Ensure that any unique or newer findings from the scattered directories are integrated into the canonical `docs/distillation/` before archiving.
4. **Update Pointers:** Ensure that high-level entry points (like `Research.md`) point directly to the canonical `docs/distillation/master_index.md`.
5. **Preserve Operational Code:** Ensure scripts (like Python extractors, parsers in `scripts/`, `knowledge_atom_extractor/`) are entirely ignored during this consolidation.

## Source Code Structure Changes
- **Create Directory:** `docs/research_archive/`
- **Move to Archive:** 
  - `distillation/` (Note: Ensure no `__init__.py` or code from this folder is strictly required for `scripts/`, if it is a Python module, keep the code but move `.md` files).
  - `distilled/`
  - `docs/distilled/`
  - Root markdown/json files: `dummy_research.md`, `gap_report.md`, `knowledge_atom_extraction_report.md`, `product_specs_report.md`, `validation_report.md`, `domain_knowledge_grouping.md`, `sdlc_phase_knowledge_mapping.md`, `master_knowledge_atoms.json`, `knowledge_atom_registry.json`, etc.
- **Modify:** `Research.md` to point to `docs/distillation/master_index.md` as the main organized distillation output.

## Data Model / API / Interface Changes
None. This task solely involves the organization of markdown documentation and JSON research artifacts.

## Verification Approach
- Ensure `docs/distillation/master_index.md` and all files referenced within it render correctly and have valid relative links.
- Verify no scripts (e.g., `*.py`, `*.sh`) or operational code artifacts have been moved or modified.
- Verify that python tests/imports still work (e.g., `pytest tests/`) if `distillation` had python files.
- Perform a manual verification by checking the root directory and `docs/` structure to confirm that the research clutter has been successfully isolated.
